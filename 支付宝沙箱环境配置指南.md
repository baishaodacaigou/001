# 支付宝沙箱环境配置指南

## 一、获取沙箱环境配置信息

1. 登录 [支付宝开放平台](https://open.alipay.com/)
2. 进入 [沙箱控制台](https://open.alipay.com/develop/sandbox/app)
3. 选择需要调试的应用，获取以下关键信息：

### 需要获取的信息：
- **App ID**：沙箱分配的应用唯一标识
- **应用私钥**：用于接口加签
- **支付宝公钥**：用于接口验签
- **支付宝网关地址**：`https://openapi-sandbox.dl.alipaydev.com/gateway.do`

## 二、配置沙箱环境

### 方法1：修改配置文件（推荐）

编辑 `server/config/alipay.js` 文件，将沙箱配置填入：

```javascript
sandbox: {
  appId: '这里填入您的沙箱AppID',
  privateKey: '这里填入您的沙箱应用私钥',
  publicKey: '这里填入您的沙箱支付宝公钥',
  gatewayUrl: 'https://openapi-sandbox.dl.alipaydev.com/gateway.do',
  notifyUrl: 'https://www.jirouai.com/api/alipay/notify',
  returnUrl: 'https://www.jirouai.com/api/alipay/return'
}
```

### 方法2：临时切换到沙箱环境

在 `server/utils/alipaySdk.js` 中修改构造函数，临时使用沙箱配置：

```javascript
// 临时使用沙箱环境（测试完成后记得改回生产环境）
constructor(config) {
  // 使用沙箱配置
  const useSandbox = true; // 设置为true使用沙箱，false使用生产环境
  
  const finalConfig = useSandbox ? config.sandbox : config;
  
  this.appId = finalConfig.appId;
  this.privateKey = this.formatPrivateKey(finalConfig.privateKey);
  this.publicKey = this.formatPublicKey(finalConfig.publicKey);
  this.gatewayUrl = finalConfig.gatewayUrl;
}
```

## 三、获取沙箱账号

1. 在沙箱控制台下载并安装 **沙箱版支付宝 App**
2. 使用沙箱账号登录，获取：
   - **商家账号**：用于模拟商家或服务商
   - **买家账号**：用于模拟用户支付行为

## 四、沙箱环境测试步骤

### 1. 配置密钥与加签方式
- 支付宝提供系统默认密钥和自定义密钥两种方式
- 若使用自定义密钥，需在平台生成并上传公钥

### 2. 进行开发与调试
沙箱环境支持大部分核心功能调试：
- ✅ 支付接口测试（支持余额支付，不支持银行卡）
- ✅ 退款与关闭订单
- ✅ 异步通知验证

### 3. 测试支付功能
- 使用沙箱买家账号登录沙箱版支付宝
- 在小程序中下单并支付
- 扫码支付需使用沙箱App的扫一扫功能

## 五、注意事项

### ⚠️ 重要提示：
1. **响应逻辑差异**：沙箱接口响应以生产环境为准，开发调试完成后必须在生产环境进行最终测试验收
2. **数据独立性**：沙箱中的用户ID、订单等数据独立于生产环境，不可混用
3. **手续费差异**：沙箱扣款手续费比例与生产环境不同
4. **数据模拟性**：返回的数据为模拟值，仅用于逻辑验证，不具备真实业务意义

### 📋 测试完成后：
1. 将配置切换回生产环境
2. 在生产环境进行最终验收测试
3. 确认所有功能正常后再上线

## 六、快速测试方案（当前可用）

如果您暂时不想配置沙箱环境，可以使用我们已经添加的**模拟支付功能**：

1. 正常点击「立即支付」
2. 当弹出「买家卖家账号相同」错误时
3. 选择「模拟支付」即可测试完整流程

这样可以快速测试订单创建、支付状态更新等核心功能，等您配置好沙箱环境后再进行真实支付测试。
